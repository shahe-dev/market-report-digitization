# Validation Notes -- 15 Enhanced JSON Files vs Template Mapping

Cross-referencing all 15 `output/*.json` files against the template mapping and HTML templates.

## Critical Issues (Will Break HTML Generation)

### 1. DAMAC Hills Apartments -- Major Structural Divergence

**File:** `damac-hills-apartments-2025-resale-report.json`

This file was generated by a different agent/prompt version and has a fundamentally different schema:

- `enhanced.investorConsiderationsEnhanced` uses `{title, factors[]}` instead of the standard 5-key structure (`entryTiming`, `configurationSelection`, `rentalStrategy`, `holdingPeriod`, `exitFlexibility`)
- `schemaOrg` uses `@type: "Report"` instead of `"Dataset"`
- `enhanced.chartSummaries` uses non-standard keys (`distributionByCategoryGraph`, `compositionSplitByEntityGraph`, `priceInsightsSummary`, `rentalInsightsSummary`, `rentalMetricsSummary`)
- `enhanced.seoMetaContent` has non-standard structure with `h2Subheadings` and `focusKeywords` instead of `h1Variations`
- Bedroom labels non-standard: "Studios", "1", "2", "3" instead of "Studio", "1 Bedroom", "2 Bedroom", "3 Bedroom"
- Component selection key names differ from all other files

**Action:** Re-run Stage 2 for this file.

### 2. DAMAC Hills Townhouses/Villas -- Empty chartSummaries

**File:** `damac-hills-townhouses-villas-2025-resale-report.json`

`enhanced.chartSummaries` is an empty object `{}`. Templates expect summary text for KPI strip, price insights table, rental insights table, completion status, and contract distribution sections.

**Action:** Re-run Stage 2 or manually populate chart summaries.

### 3. Arabian Ranches 3 -- sectionOrder/componentSelection Mismatch

**File:** `arabian-ranches-3-townhouses-2025-resale-report.json`

sectionOrder contains `completionStatus` but `componentSelections.completionStatus` maps to `stacked-bar-dual`. The rendering engine will try to render indicators-block (because sectionOrder says `completionStatus`) but the data is structured for a stacked-bar-dual.

**Action:** Either change sectionOrder entry to `completionStatusDual`, or change the component selection to `indicators-block`.

## Moderate Issues (Graceful Handling Required)

### 4. Null executiveSummary at Root Level

**Files:** `business-bay-apartments`, `city-walk-apartments`

Root-level `executiveSummary` is null. Only `enhanced.executiveSummaryEnhanced` is populated. The template uses `enhanced.executiveSummaryEnhanced` so this is fine -- but flagged for awareness.

### 5. Null grossRentYield

**Files:** `damac-lagoons-townhouses` (null), `the-valley-townhouses` (null)

Legitimate data gaps (emerging communities with nascent rental markets). The KPI strip template hardcodes a "Gross Rental Yield" card -- rendering engine must handle null gracefully (display "N/A" or similar).

### 6. Missing Top-Level transactionGrowth and valueGrowth

**Files:** ALL 15

No file has top-level `transactionGrowth` or `valueGrowth` fields. Some files embed these inside `keyPerformance` as non-standard extra fields:
- `keyPerformance.resaleTransactionGrowth` (DAMAC Hills 2, DAMAC Hills TV, DAMAC Lagoons, District One)
- `keyPerformance.resaleTransactionsGrowth` (Business Bay)
- `keyPerformance.transactionGrowth` (DAMAC Hills Apartments)

**Template impact:** The KPI strip template uses `{{IF transactionGrowth}}` and `{{IF valueGrowth}}` -- these will always be falsy at root level.

**Resolution for developer:** Look for growth values in `keyPerformance.resaleTransactionGrowth` or `keyPerformance.resaleTransactionsGrowth` as fallbacks. If neither exists, omit the YoY badge on those KPI cards.

## Low-Priority Issues (Inconsistencies, Not Blockers)

### 7. contactDistribution Typo

All files use `contactDistribution` (not `contractDistribution`). This is consistently misspelled across the entire dataset. The templates already reference `contactDistribution` to match.

### 8. Non-Standard componentSelection Key Names

**Impact:** The rendering engine does not need these keys to select templates -- it uses `sectionOrder` entries. These keys are informational/diagnostic only.

### 9. Dubai Market Sub-Splits

**Files:** `arabian-ranches-townhouses`, `arabian-ranches-villas`, `damac-hills-2-townhouses-villas`, `damac-lagoons-townhouses`

These files have `dubaiSalesSplit` and/or `dubaiRentalSplit` in componentSelections. Per the SECTION-TEMPLATE-MAP conditional branching rules, these should render additional stacked bars before the main Dubai market comparison cards. However, no corresponding sectionOrder entries exist for these sub-splits -- they appear to be diagnostic metadata rather than rendering instructions.

**Action for developer:** Ignore unless these appear in sectionOrder.
