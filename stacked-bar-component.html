New Component: Stacked Bar

8.4 STACKED BAR (PERCENTAGE MODE) ID: stacked-bar

Horizontal stacked bar for part-to-whole visualization. Supports two modes: percentage display (showing %) or absolute value display (showing AED amounts with proportional bar widths).

Transactions by Completion Status

Off-plan transactions dominated The Valley townhouse market at 69.3% of total volume, with ready properties comprising the remaining 30.7%. This distribution reflects the community's development stage and continued buyer confidence in under-construction inventory.

Source: Property Monitor

8.5 STACKED BAR DUAL (TWO RELATED DISTRIBUTIONS) ID: stacked-bar-dual

Use when showing two related distributions side by side with absolute values. Bar widths are calculated from ratios; percentages shown inside bars in bold white, absolute values in legend.

Completion Status: Transactions vs Value

Off-plan properties accounted for 461 of 665 total transactions (69.3%) and AED 1.54B of AED 2.14B total value (71.9%), indicating slightly higher average transaction values for off-plan units.

Source: Property Monitor

STACKED BAR - ABSOLUTE VALUES MODE ID: stacked-bar

When input is absolute values (not percentages), calculate bar widths from the ratio of each value to the total. Display the actual values as labels, not percentages. Example: AED 1.54B + AED 0.60B = AED 2.14B total. Bar widths: 71.9% (1.54/2.14) and 28.1% (0.60/2.14).

Resale Value by Completion Status

Off-plan resale transactions generated AED 1.54 billion while ready properties accounted for AED 0.60 billion, totaling AED 2.14 billion in resale value. The visual proportion reflects the 71.9% to 28.1% value split.

Source: Property Monitor

⚠️ EXTREME RATIO (>90%) — DO NOT USE STACKED BAR

When one category exceeds 90%, the stacked bar becomes visually meaningless (just a sliver). Instead, use an Indicators Block highlighting the dominant category.

Dubai Hills Estate: 99.5% Ready

Dubai Hills Estate's villa market is dominated by ready resale properties at 99.5%. When ratios are this extreme, an indicators block communicates more effectively than a stacked bar.

Component Selection Condition

Condition for Stacked Bar:


Use when presenting two values as part-to-whole composition where:

  • Exactly 2 categories (off-plan/ready, new/renewal, etc.)
  • Visual representation of proportion is desired
  • Input can be percentages OR absolute values
  • Neither category exceeds 90%

Mode Selection:

PERCENTAGE MODE:
  Input: 69.3%, 30.7%
  Bar widths: 69.3%, 30.7%
  Labels: "69.3%", "30.7%"

ABSOLUTE VALUE MODE:
  Input: AED 1.54B, AED 0.60B
  Total: 1.54 + 0.60 = 2.14
  Bar widths: (1.54/2.14)=71.96%, (0.60/2.14)=28.04%
  Labels: "AED 1.54B", "AED 0.60B"

Selection Logic:

IF data_type == "two_category_split"
   AND max(value1, value2) <= 90%
   AND visual_proportion_desired == true
THEN use Stacked Bar

IF max(value1, value2) > 90%
THEN use Indicators Block showing dominant category
     Example: "99.5% Ready" in primary indicator card

IF comparing same split across multiple entities
THEN use Stacked Column instead

IF visual_representation_desired == false
THEN use Comparison Metrics Cards instead

Use stacked-bar-dual when: Showing two related splits side-by-side (e.g., transactions AND value)


⚠️ Do NOT use Stacked Bar when:

  • One category exceeds 90% → use Indicators Block (e.g., "99.5% Ready")
  • More than 2 categories → use Stacked Column or Data Table
  • Comparing same ratio across multiple entities → use Stacked Column
  • Exact values matter more than visual proportion → use Comparison Metrics Cards
